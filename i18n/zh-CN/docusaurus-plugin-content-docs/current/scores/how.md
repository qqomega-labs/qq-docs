## 如何解读分数

QQ 会为每个维度（基本面、代币经济、链上、技术、宏观）分别给出 <span className="pink-glow-text">**0–100 分**</span>，并为每个项目计算一个 **QQ 总评分**。  
这些分数应被理解为“质量相对于估值 recognised 的程度”，而不是绝对的买卖信号。

一般解读如下：

| 分数区间 | 标签 | 含义 |
|:------:|:----:|:-----|
| 🔵 80 – 100 | 显著低估 | **分数显著高于估值所反映的水平** → 强烈低估 |
| 🟢 60 – 80  | 偏低估 | **分数高于估值所反映的水平** → 低估 |
| 🟡 40 – 60  | 估值合理 | **分数与估值大体匹配** |
| 🟠 20 – 40  | 偏高估 | **分数低于估值所反映的水平** → 高估 |
| 🔴 0 – 20   | 显著高估 | **分数远低于估值所反映的水平** → 极度高估 |

### QQ 里的“低估 / 高估”到底是什么意思？

在 QQ Omega 中，“低估 / 高估”来自一个相对估值过程：  
系统会把资产的 <span className="pink-glow-text">当前估值</span> 与各集群评分所隐含的 <span className="pink-glow-text">质量水平</span> 进行对照，然后将这些视角汇总为一个 <span className="pink-glow-text">**QQ 总评分**</span>，用来表达该资产在“质量—估值”关系上更像 <span className="pink-glow-text">**低估**</span> 还是 <span className="pink-glow-text">**高估**</span>。

举例来说：当一个资产在基本面、链上行为、代币经济与结构信号上都较强，但市场估值尚未充分反映这份质量时，它往往会落在“低估区间”。相反，如果质量信号偏弱，但估值仍处于高位（通常由叙事、热度或短期投机推动），它更可能落在“高估区间”。

QQ 会把这些“质量与估值的不匹配”显性化，帮助你在条件更有利时考虑 <span className="pink-glow-text">**逐步布局**</span>，或在条件过热时选择 <span className="pink-glow-text">**止盈 / 降风险**</span>。
