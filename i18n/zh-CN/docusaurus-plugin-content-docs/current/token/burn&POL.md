### 运作方式

协议金库收到的 LP 手续费，会按照以下分配规则进行处理。

### 销毁分配（回购并销毁）

<span className="pink-glow-text">**LP 手续费的 12%**</span> 会进入 $QQ 的 <span className="pink-glow-text">**销毁机制**</span>。  
这部分资金用于在市场上回购 $QQ，并将代币永久发送至销毁地址（dead address），从而在时间维度上持续减少流通供应量。

### POL 分配（协议自有流动性）

<span className="pink-glow-text">**LP 手续费的 17%**</span> 会进入 <span className="pink-glow-text">**POL（Protocol-Owned Liquidity，协议自有流动性）**</span>。  
POL 的含义是：协议使用手续费把 $QQ 与配对资产一起补充为流动性，从而长期支持 **Solana** 与 **Base** 上的 $QQ 流动性池。

### 销毁规则

销毁过程会持续进行，直到总供应量降至 <span className="pink-glow-text">**???.???.???**</span> 枚 $QQ。  
最终供应量并非预先固定，它将随协议长期活跃度而变化；但该机制的设计目标是长期 <span className="pink-glow-text">**至少销毁初始总供应量的 50%**</span>。

### 月度执行与透明披露

协议分配将以 <span className="pink-glow-text">每月</span> 为周期，通过链上交易透明执行，并汇总上一周期的全部活动。  
每个月，QQ Omega 都会发布公开回顾，并附上每一项协议级资金操作的 **链上凭证**，任何人都可以独立核验资金如何被使用、转移，或从流通中移除。
