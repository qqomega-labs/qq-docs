## 评分更新频率

时机很重要。

QQ Omega 为不同数据流设置了不同的更新频率，并根据各类数据的天然特性进行校准。  
链上数据会尽可能接近实时处理；而价格行为、成交量、订单簿等市场数据则以日内或分钟级间隔刷新。链下输入与基本面信息（例如团队动态、文档更新、法律或监管披露）会在官方渠道发布新信息后被纳入系统。

在早期实现阶段，系统可能会为了简化而依赖聚合型数据服务商；但长期目标是尽可能贴近 <span className="pink-glow-text">实时数据源</span>。维护一条“新鲜、稳健、尽量靠近源头”的数据管道是 <span className="pink-glow-text">核心优先级</span>：任何在质量或延迟上出现退化的数据源都会被持续复审，并在必要时替换。

## 质量校验与反垃圾过滤

为了让评分保持干净且可信，QQ 在任何数据影响智能体输出或最终评分之前，都会执行分层验证流程。系统会持续检查输入数据是否存在缺失值、不可能的数值、重复记录；并使用统计方法识别异常模式，例如人为成交量尖峰、刷量交易、或由机器人驱动的活动。只要条件允许，信息会在多个独立数据提供方之间进行交叉核对，以降低单一来源带来的偏差。

同时，QQ 还会应用一套“反操纵”启发式规则，用于识别那些目标仅仅是抬高评分、而非反映真实活跃度的行为，例如短期注入流动性、或伪装成自然交易的刷量活动。

只有通过这些 <span className="pink-glow-text">质量校验</span> 后，数据才会被允许在智能体体系中向下游传播并影响评分。一旦检测到异常，相关输入可能会被 <span className="pink-glow-text">标记</span>、<span className="pink-glow-text">降低权重</span> 或 <span className="pink-glow-text">剔除</span>，以保护系统的整体可信度与一致性。
